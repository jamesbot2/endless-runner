# Building & Packaging

## Quick reference

| Command | What it does | Output |
|---|---|---|
| `npm run dev` | Vite hot-reload + Electron (development) | — |
| `npm run build` | Compile TypeScript + Vite bundle | `dist/` |
| `npm run pack` | Build + `electron-builder --dir` | `release/win-unpacked/` |
| `npm run dist` | Build + NSIS installer | `release/Subway Surfer Setup *.exe` |

From repo root, prefix with `npm --prefix apps/desktop` or use the shorthand:

```bash
npm run desktop:dev      # ≡ npm --prefix apps/desktop run dev
npm run desktop:build    # ≡ npm --prefix apps/desktop run build
npm run desktop:pack     # ≡ npm --prefix apps/desktop run pack
npm run desktop:dist     # ≡ npm --prefix apps/desktop run dist
```

---

## 1. Development

```bash
npm run desktop:dev
```

Starts Vite dev server (`:5173`) and launches Electron with DevTools.
Game files are served from memory — no build step needed for frontend changes.

---

## 2. Build

```bash
npm run desktop:build
```

Compiles:
- `electron/main.ts` + `electron/preload.ts` → `dist/electron/` (tsc)
- `src/` renderer → `dist/renderer/` (Vite)

Output goes to `apps/desktop/dist/`.

---

## 3. Package (win-unpacked)

```bash
npm run desktop:pack
```

Generates `apps/desktop/release/win-unpacked/Subway Surfer.exe` — a
**portable, no-installer** directory. No symlink privileges required.
Run the `.exe` directly from that folder.

> `pack` passes `--config.win.signAndEditExecutable=false`, which skips
> `winCodeSign` entirely — no symlink issue, no code signing, no icon/ resource
> editing. Suitable for local testing and CI.
> 
> For production releases, use `desktop:dist` and handle signing separately.

---

## 4. Distributable Installer (NSIS)

```bash
npm run desktop:dist
```

Generates `apps/desktop/release/Subway Surfer Setup x.x.x.exe` — a
Windows NSIS installer.

### ⚠️ Symlink / signing privilege note

`electron-builder` calls `winCodeSign` during the NSIS step (`desktop:dist`),
which needs **Create Symbolic Links** permission. On Windows you may see:

```
Cannot create symbolic link，客户端没有所需的特权
```

**Fix options (pick one):**

1. **Enable Windows Developer Mode**
   *Settings → Update & Security → For developers → Developer Mode*
   (Automatically grants symlink permission for the current user.)

2. **Run as Administrator**
   ```powershell
   npm run desktop:dist
   ```

3. **Use `desktop:pack` instead** — skips `winCodeSign` entirely
   (`--config.win.signAndEditExecutable=false`), so no symlink needed.
   Generates `release/win-unpacked/` — fully functional, just no installer.

### When to use each

| Command | Use case | Signing | Symlink needed |
|---|---|---|---|
| `desktop:pack` | Local testing, CI, quick builds | ❌ Skipped | ❌ No |
| `desktop:dist` | Production installer, public release | ✅ Can sign | ✅ Yes |

> For a release build, run `desktop:pack` on a dev machine for testing,
> then `desktop:dist` on a CI runner or signed build environment that has
> both symlink permission and a code signing certificate.

## Files not committed

- `node_modules/` — install via `npm ci` in CI or `npm install` locally
- `dist/` — build artifacts (generated by `build`)
- `release/` — electron-builder output (generated by `pack` / `dist`)
